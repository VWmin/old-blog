2.3.1记忆化搜索与动态规划

引入：背包问题

首先问题可用基础的dfs解决。缺点是重复搜索了一些情况，浪费了时间。
提出：记忆化搜索

实现：引入一个二维数组（相对于该二参函数），通过这个数组记录函数参数组合的返回值。每次调用函数时，如果参数组合已经过处理则通过该数组直接返回值。

缺陷：参数过多时难以实现

简化：同样使用这样的数组，并用二重循环替代递归。

理解：这种方法通过一个（一些）初始值按照某种递推式一步步得到结果。

tip：动态规划按照不同的递推式可正向可逆向。

最长公共子序列问题

（对书上内容的直接理解）

将需要比较的s、t字符串划分为s[m]、t[n]。二重循环两两比较两个字符数组中的字符。然后搞事。

（可能是抽象化理解 我也不知道对不对。。）

两个循环遍历出所有的子串组合。比如s为’a’，t为’b’、‘bc’、‘bcd’、‘bcde’。每次比较后将结果传递给当前比较位置（i，j）各加一的位置（i+1, j+1（这叫下一轮比较吧？））。因为要求最长公共子串，当没有比较到相同字符时传递下一次比较（【i+1， j】、【i， j+1】）中的最大值。

递推分析：

定义，dp[i+1][j]为选择前i个物品总价值为j时的最小重量

第i+1个物品和为j时，则有，

  • 前i个物品和为j

  • 前i个物品和为j-v[i+1]

初始化，根据定义，当i=0时表示没有选择物品，j=0则价值为零重量为零dp[0][0] = 0; j≠0则是不存在的情况，设为足够大的值d[0][0] = INF;

多重部分和问题

第一种
定义，dp[i+1][j]为前i个数加和是否能为j

所以，当前i个数加和为j时，前i-1个数加和至少为k*a[i]中的一个

那么，E k∈{0≤k≤m[i] 且 k*a[i]≤j} 使得 dp[i+1][j] = dp[i][j – k*a[i] ]

第二种

定义，dp[i+1][j]为前i个数加和为j时，第i个数剩余的最少个数（为什么是最少？当最少剩余≥0时才说明正好能组成数j或还有余豁）

那么，dp[i+1][j] = {

  m[i], ( 前i-1个数得到了j )

  -1， ( j < a[ i ] || dp[ i+1 ][ j-a[i] ] < 0 )

  dp[ i+1 ][ j-a[i] ] - 1，( 不是很懂 )

 }

最长上升子序列问题

定义，dp[i+1]为长度为i的序列的最长上升子序列的长度

  那么，dp[i+1] = {

  dp[i], (a[i] > a[i+1])

  temp=dp[i] + 1, (temp<= a[i+1])

}

2.3.3 有关计数问题的DP

划分数

（DP不仅对于求解最有问题有效，对于各种排列组合的个数、概率或者期望之类的计算同样很有用）

定义，dp[ i ][ j ]为 j 的 i 划分总数

对于数n,有n的m划分为：

$$\sum_{i=1}^Na_i = n$$

则有n-m的m划分为：

$$\sum_{i=1}^N(a_i - 1) = n - m$$

当存在ai = 0时，有n的m-1划分（比如1+0+3是4的2划分）：

$${\sum_{i=1}^N(a_i - 1) = n - m, \exists i \in [1, m] \Rightarrow a_i = 0}$$

所以有递推式：dp[i][j] = dp[i][j-i] + dp[i-1][j]

记得初始化dp[0][0] = 1;（虽然不知道为什么）

多重集组合数

定义，dp[i+1][j]为前i种物品取j个的取法

又，从前i种物品取j个可以先从前i-1种物品取j-k个，再从第i种物品取k个，所以：

$$dp[i+1][j] = \sum_{k=0}^{\min(a[i], j)}dp[i][j - k]$$
